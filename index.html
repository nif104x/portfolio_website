Invoice OCR & Table Parsing (Production)

Problem: Automate invoice processing across heterogeneous vendors with noisy scans and multiple languages.

Data: 200k scanned docs • mixed languages • Stack: PyTorch, Detectron2, Transformers, Tesseract, Docker, AWS
Open full case study — Problem, Data, Methodology, Results, Impact, Code

1. The Problem

High-volume invoice intake required robust extraction of fields and table elements to feed accounting systems. Failure modes: rotated scans, multi-column layouts, handwritten annotations.

2. The Data

  • Source: client document pipeline + synthetic generation (30% synthetic)
  • Size: 200,000 images. Training/val split: 160k/20k
  • Preprocessing: deskew, contrast normalization, OCR token alignment, bounding-box standardization
  • Feature engineering: region-proposals + token positional encodings for table parsing

3. Methodology

Approach: two-stage pipeline

  1. Layout detection: Detectron2 Faster R-CNN baseline → fine-tuned with focal loss for imbalanced boxes.
  2. Table parsing: Transformer-based relation classifier on tokens extracted by OCR; multi-task loss for cell segmentation + cell relation prediction.
  3. Postprocessing: rule-based normalization and confidence calibration (Platt scaling).

4. Results & Evaluation

Key metrics (test):
  • Field extraction F1: 0.93
  • Table cell IoU: 0.89
  • End-to-end accuracy (key fields): 0.91
  • Throughput: 12 pages/sec per GPU (batching)

5. Impact

Automated processing reduced manual efforts by 87%, cut invoice processing time from avg 36 hours to <6 hours, and reduced error-rate in GL posting by 4 percentage points.

6. Code & Repro

Repository (anonymized): github.com/yourname/invoice-ocr

Architecture (summary)

  • Detectron2 Faster R-CNN → box proposals
  • ResNet50 backbone, FPN
  • Transformer relation head (6 layers, 8 heads)
  • Loss: CE + IoU + relation-binary cross-entropy

Hyperparams

lr=3e-5 · batch=8 · weight_decay=0.01 · warmup=500 · epochs=18

Notes & pitfalls

Synth data required for rare layouts. Balanced sampling and focal loss improved small object recall. Use confidence thresholds per vendor for production routing.
